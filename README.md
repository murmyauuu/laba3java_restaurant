# Многопоточная система работы ресторана (официанты+кухня)

Лабораторная работа представляет собой консольную многопоточную симуляцию функционирования ресторана, реализованную на языке программирования Java.

Целью работы является демонстрация принципов многопоточного программирования, синхронизации потоков и корректной организации взаимодействия между параллельно выполняющимися процессами без возникновения состояний гонки и взаимных блокировок.

---

## 1. Архитектура системы

Программная система состоит из следующих основных компонентов:

- **`Main`** — точка входа в программу, инициализация официантов и кухни.
- **`Waiter`** — официант, работающий в отдельном потоке: принимает заказы, отправляет их на кухню и доставляет готовые блюда клиентам.
- **`Chef`** — задача повара, выполняемая в пуле потоков: имитирует приготовление блюда с заданной задержкой.
- **`Kitchen`** — центральный компонент кухни, управляющий очередью заказов и пулом поваров.
- **`Order`** — модель заказа, содержащая идентификатор, тип блюда и ссылку на официанта.
- **`DishType`** — перечисление доступных блюд с указанием времени приготовления в миллисекундах.
- **Логирование** — осуществляется через `System.out.println()`.

---

## 2. Логистика обработки заказов

Обработка заказов осуществляется по следующему алгоритму:

1. Официант (в своём потоке) генерирует заказ (имитация приёма от клиента).
2. Заказ помещается в **общую очередь кухни**.
3. Кухня, работающая в фоновом потоке, извлекает заказы из очереди и передаёт их **пулу поваров**.
4. Повар начинает приготовление: `Thread.sleep()` имитирует время готовки.
5. По завершении приготовления:
    - Заказ помечается как готовый;
    - Официант, ожидавший через `synchronized (order) { wait(); }`, получает уведомление через `notifyAll()`.
6. Официант доставляет блюдо клиенту.

Все операции с очередью и синхронизацией выполнены с использованием механизмов, изученных в курсе:
- `BlockingQueue` — для потокобезопасной очереди;
- `synchronized` + `wait()`/`notifyAll()` — для ожидания готовности;
- `ExecutorService` — для управления пулом поваров;
- `volatile` — для безопасного флага готовности.
